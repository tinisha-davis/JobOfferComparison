# Contributing Guidelines


## Submitting Changes 

See something you want to improve/a bug that needs to be fixed?

There are 2 ways you can contribute to our project, as Student in the Project Management in Software Development course at NSC, or as an Open Source contributor

### To Submit Changes as a Student:
1. Clone the repository
2. Create a new branch:

    ```bash
    git checkout -b your_name_feature_your_working_on
    ```

3. Make your changes
4. Commit your changes 
    ```bash
    git commit -m 'Add feature'
    ```
5. Push to the branch: 
    ```bash
    git push origin feature-branch
    ```
6. Create a pull request
7. Don't merge your PR unless someone else's ticket is blocking someone else's ticket
8. Don't delete your branch after merging

### To Submit Changes as an Open Source Contributor:

1. Clone the repository
2. Create a new branch: 'git checkout -b feature-branch'
3. Make your changes
4. Commit your changes 'git commit -m 'Add feature''
5. Push to the branch: 'git push origin feature-branch
6. Create a pull request
7. Communicate that you've created a pull request
